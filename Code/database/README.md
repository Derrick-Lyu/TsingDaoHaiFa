# Database

This directory owns PostgreSQL bootstrap and schema evolution assets for the Qingdao HaiFa platform.

## Responsibilities

- `init/`: first-run bootstrap SQL mounted into the Postgres container
- `migrations/`: versioned schema changes for application-owned tables
- `seeds/`: development or demo seed data

## Current Demo Schema

The first demo migration lives in `migrations/001_fund_safety_demo_schema.sql` and defines the minimal schema for the fund safety / terror risk workflow.
The reviewer-assignment extension lives in `migrations/002_terror_alert_review_assignments.sql` and adds single-alert manual assignment fields directly onto `terror_alert_reviews` so assignment stays aligned with the existing review flow.

Tables and responsibilities:

- `fund_safety_topic_summaries`: executive summary rows for each `资金安全` secondary topic
- `terror_blacklist`: maintainable blacklist entries for terror-risk screening
- `terror_rules`: rule definitions and enable/disable state
- `terror_rule_params`: editable thresholds and rule parameters
- `payment_transactions`: transaction-level input data for demo detection runs
- `terror_detection_jobs`: job execution history and result counters
- `terror_alerts`: risk findings generated by a detection job
- `terror_alert_evidences`: evidence records used by the alert detail page
- `terror_alert_reviews`: manual review status, notes, and disposition
  - also stores v1 assignment state via `assignment_status`, `assigned_reviewer_name`, and `assigned_at`

## Migration And Seed Expectations

- Keep bootstrap scripts idempotent.
- Add new schema changes as versioned migrations instead of editing old SQL in place.
- Keep demo seed data in `seeds/` and make it business-aligned with Qingdao HaiFa.
- Seed data should support the summary page, the terror-risk topic page, the CRUD pages, and the three demo rule types.
- Prefer storing source data in tables and letting the backend generate alerts through the detection job flow.

## Notes

- Domain schema is intentionally deferred until backend APIs are defined, but the demo schema now covers the core workflow needed for the fund safety / terror risk prototype.
